DATABRICKS SQL SETUP INSTRUCTIONS
=====================================

This guide will help you load the SmartLogix transport optimization data into Databricks SQL.

STEP 1: PREPARE YOUR DATABRICKS WORKSPACE (FREE EDITION)
--------------------------------------------------------

1. Access your Databricks workspace (free edition)
2. Navigate to "SQL Warehouses" in the left sidebar
3. Create a new SQL Warehouse:
   - Name: "smartlogix-warehouse"
   - Cluster size: "2X-Small" (free tier limit)
   - Auto-stop: 10 minutes (to save credits)
   - Enable serverless: No (not available in free tier)
   - Note: Free tier has limited compute hours per month

STEP 2: UPLOAD RAW DATA FILES (FREE EDITION)
---------------------------------------------

IMPORTANT: Free edition has file size limits (typically 2GB per file)

1. Go to "Data" in the left sidebar
2. Click "Create" → "Table"
3. Choose "Upload File" option
4. Upload each CSV file from data/raw/ folder:
   - delhivery.csv (may be large - consider splitting if >2GB)
   - distance.csv
   - order_small.csv
   - order_large.csv
   - supply_chain_data.csv
   - Inventory Data.xlsx
   - Model Sample Data - Retail Inventory.xlsx

5. For each file:
   - Choose "Upload to DBFS" (Unity Catalog not available in free tier)
   - Set destination path: /FileStore/shared_uploads/smartlogix/
   - Note: Free tier has limited storage space

STEP 3: CREATE DATABASE AND SCHEMA (FREE EDITION)
--------------------------------------------------

1. Go to "SQL Editor" in Databricks
2. Create database and schema (free edition uses default catalog):

```sql
-- Create database (free edition uses default catalog)
CREATE DATABASE IF NOT EXISTS smartlogix;

-- Use the database
USE smartlogix;

-- Create schema for tables
CREATE SCHEMA IF NOT EXISTS transport_data;
```

STEP 4: RUN DATA CLEANING SCRIPT
--------------------------------

1. Upload the data cleaning script to your workspace
2. Run the script to process raw data:

```python
# Run the data cleaning script
python data_cleaner.py
```

This will create cleaned CSV files in data/processed/ folder.

STEP 7: CREATE ANALYTICS VIEWS
------------------------------

```sql
-- Transport Summary View
CREATE OR REPLACE VIEW smartlogix.transport_data.transport_summary AS
SELECT 
    source_center,
    destination_center,
    COUNT(*) as trip_count,
    AVG(actual_time) as avg_actual_time,
    AVG(osrm_time) as avg_osrm_time,
    AVG(factor) as avg_factor,
    AVG(actual_distance_to_destination) as avg_distance
FROM smartlogix.transport_data.delhivery_data 
GROUP BY source_center, destination_center;

-- Order Summary View
CREATE OR REPLACE VIEW smartlogix.transport_data.order_summary AS
SELECT 
    source,
    destination,
    order_size,
    COUNT(*) as order_count,
    SUM(weight) as total_weight,
    AVG(EXTRACT(EPOCH FROM (deadline - available_time))/3600) as avg_hours_to_deadline
FROM smartlogix.transport_data.order_data 
GROUP BY source, destination, order_size;

-- Dashboard Analytics View
CREATE OR REPLACE VIEW smartlogix.transport_data.dashboard_analytics AS
SELECT 
    'Total Transport Records' as metric,
    (SELECT COUNT(*) FROM smartlogix.transport_data.delhivery_data) as value
UNION ALL
SELECT 
    'Total Orders',
    (SELECT COUNT(*) FROM smartlogix.transport_data.order_data)
UNION ALL
SELECT 
    'Total Routes',
    (SELECT COUNT(DISTINCT CONCAT(source_center, '->', destination_center)) 
     FROM smartlogix.transport_data.delhivery_data)
UNION ALL
SELECT 
    'Average Efficiency',
    (SELECT ROUND(AVG(factor), 2) 
     FROM smartlogix.transport_data.delhivery_data 
     WHERE factor IS NOT NULL);
```

TROUBLESHOOTING
---------------

1. **File Upload Issues:**
   - Ensure files are in CSV format
   - Check file size limits (max 2GB per file)
   - Verify file paths are correct

2. **Data Type Issues:**
   - Use the COPY INTO command with 'inferSchema' = 'true'
   - Check for special characters in data
   - Verify timestamp formats

3. **Permission Issues:**
   - Ensure you have CREATE TABLE permissions
   - Check database and schema access rights
   - Verify Unity Catalog permissions if using Unity Catalog

4. **Performance Issues:**
   - Use appropriate cluster size for large datasets
   - Consider partitioning large tables
   - Use Delta format for better performance
